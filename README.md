# CallMissed — Hiring Assignments

Welcome 👋 This repository contains the take-home assignments for the roles we're hiring for at **CallMissed**. Pick the assignment that matches the role you applied for, build it, and submit it.

CallMissed is an AI-native platform: one OpenAI-compatible API for 300+ LLMs, image generation, speech-to-text, text-to-speech, and voice agents. These assignments are intentionally close to the kind of work you'd actually do here.

## ✅ The rules (read this first)

1. **AI tools are 100% allowed — and encouraged.** Use Cursor, Claude, ChatGPT, Copilot, v0, whatever you want. We build with AI every day; we expect you to. There is **no penalty** for using AI. Using it well is a skill we're hiring for.
2. **Everything will be analyzed.** We review the full repo: code quality, structure, commit history, README, how you handled edge cases, security (no leaked keys), and product taste. "It works" is the floor, not the ceiling.
3. **Your README must be updated.** Replace/extend the README in *your* submission repo to explain: what you built, how to run it, the decisions and trade-offs you made, what you'd do with more time, and (honestly) where you used AI and how.
4. **Attach an updated résumé.** Commit your current résumé (PDF) to your submission repo as `resume.pdf` (or link it clearly in your README).
5. **Your own public repo.** Build in a **public** GitHub repository under *your* account (not a private one, not a PR to this repo).

## 📤 How to submit

When you're done, **email the link to your public repo** to **[karan@callmissed.com](mailto:karan@callmissed.com)** with:

- **Subject:** `Assignment — <Role> — <Your Name>` (e.g. `Assignment — Next.js — Asha Rao`)
- **Body:** the public repo URL, a live demo URL if you deployed one, and ~3 lines on your approach.
- Make sure `resume.pdf` and an updated `README.md` are in the repo.

We read every submission. If it's good, you'll hear from us fast.

---

## ⏱️ Time & scope

- Target **6–10 focused hours**. We are **not** looking for a finished product — we're looking for how you think, build, and communicate.
- Ship something working and well-explained over something huge and broken. A small, polished, well-documented build beats a sprawling unfinished one.
